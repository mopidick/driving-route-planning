# UI 体验迭代日志

## Round 1 - 基线评审（subagents 交叉）

日期：2026-03-22

### 评审分数
- R1-A（信息架构/任务流）：66/100（达标线 82）
- R1-B（视觉/动效）：72/100（达标线 82）
- R1-C（工程可实现性）：68/100（Round2预期 79+）

### 关键问题
- 抽屉仅按钮切态，无手势连续性
- 状态分散，缺统一 UI 状态机
- 地图与抽屉联动不足，full态误触风险
- 卡片主次不清，推荐解释不足
- 底部 tab 与浮层层级冲突风险

### 执行改造
- 新增全局 UI 状态：`sheetLevel/detailOpen`
- `BottomSheet` 增加拖拽吸附三态
- 推荐页 full 态锁定地图交互并降 marker 密度
- 同页快速参数调整（mode/hours）
- 卡片改为主CTA采纳 + 次级详情 + 已采纳态
- 新增推荐因子标签
- 首页定位状态 pill，初始折叠态

## Round 2 - 复审（subagents）

日期：2026-03-22

### 复审分数
- R2-A：84/100（已达标）
- R2-B：85/100（已达标）
- R2-C：81/100（接近达标，建议再补工程稳态）

### 剩余高影响项
- 手势鲁棒性：touch-only，需 pointer 统一
- 快速调参缺去抖，可能请求风暴
- 地图重绘抖动：需要进一步减少全量拟合
- 抽屉与地图滚动边界需更稳定

### Round 2.5 补强（已实施）
- BottomSheet 从 touch 迁移到 pointer 事件
- 推荐页 mode/hours 改为 260ms 去抖重算
- 地图 marker 密度改为分布式抽样（非简单截断）
- EmbeddedMap 减少重复 fitView 触发

## Visual Round 1 - 方向评审（subagents）
